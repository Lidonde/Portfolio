# 100Hires Portfolio Project

This project documents my process of setting up an AI-assisted development environment using Cursor IDE and integrating key extensions. It highlights my ability to learn new tools independently, troubleshoot issues, and clearly document my workflow.

## Tools Installed

- Cursor IDE – AI-native code editor
- Claude Code extension – for AI-assisted coding support
- Codex extension – for code generation and logic assistance
- GitHub – for version control and project hosting

## Steps Completed

1. Installed Cursor IDE from the official website  
2. Opened the IDE and explored the interface  
3. Installed Claude Code extension and completed login  
4. Installed Codex extension and authenticated successfully  
5. Created a public GitHub repository  
6. Opened the repository in Cursor  
7. Created and edited the README.md file  
8. Staged, committed, and pushed changes to GitHub  

## Issues Encountered & Solutions

- **Authentication delays during extension login**  
  *Solution:* Retried login and ensured browser pop-ups were enabled  

- **Initial confusion with Git workflow (staging vs committing)**  
  *Solution:* Followed a GitHub tutorial in YouTube and used `git add .` before committing  

- **Multiple AI suggestions appearing simultaneously**  
  *Solution:* Adjusted extension usage to focus on one tool at a time  

## Key Learnings

- Learned how to set up and navigate a new AI-powered IDE  
- Gained hands-on experience with GitHub repositories and version control  
- Improved my ability to troubleshoot setup issues independently  
- Strengthened my documentation and structured thinking skills  

## Conclusion

This project demonstrates my ability to quickly adapt to new tools, solve problems independently, and document my process in a clear and structured way.

# LinkedIn Organic Content Strategy for B2B SaaS

This repository compiles **10 high-signal experts** across LinkedIn, YouTube, and podcasts who actively practice **B2B SaaS marketing, demand generation, and organic content strategy**.  

The goal is to extract **real-world, actionable insights** that can later inform a **practical LinkedIn-led growth playbook**.

## Research Methodology

1. **Topic Selection**  
   - Chosen topic: **LinkedIn Organic Content Strategy for B2B SaaS**  
   - Criteria: relevance to pipeline-driven growth, demonstrated LinkedIn expertise, and ability to translate strategy into action.

2. **Expert Identification**  
   - Selected practitioners who actively use LinkedIn and other channels to drive measurable results.  
   - Evaluated based on recent, high-signal content, diversity of expertise (ABM, positioning, storytelling, growth systems), and ability to share actionable tactics.

3. **Content Collection**  
   - **LinkedIn posts:** manually curated to preserve exact content and posting dates.  
   - **YouTube transcripts:** collected via YouTube API for completeness and accessibility.  
   - **Podcasts:** analyzed for frameworks, lessons, and actionable tactics.  
   - Each source annotated with **insights and potential application** to B2B SaaS content strategy.

4. **Repository Organization**  
   - `/research/sources.md` — list of all experts with links, dates, and brief annotations  
   - `/research/linkedin-posts/` — posts organized by author  
   - `/research/youtube-transcripts/` — transcripts organized by video  
   - `/research/other/` — additional materials, including podcast summaries
